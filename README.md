# Auto Refresh Pro

Auto Refresh Pro 是一款基于 Chrome Extension Manifest V3 的网页自动刷新插件，支持定时全页刷新、XPath 点击刷新、页面区域变化监控、系统通知提醒、网页悬浮控制窗和运行日志倒计时。

当前版本：`1.0.6`

## 功能特性

- 定时全页刷新：按秒设置刷新间隔，适合行情页、列表页、监控页等需要自动更新的场景。
- XPath 点击刷新：选取页面元素后定时模拟点击，适合“加载更多”“刷新按钮”“分页按钮”等局部刷新场景。
- iframe 支持：XPath 选取和点击支持 iframe 场景，插件会记录目标所在框架。
- 最大刷新次数：可设置刷新上限，`0` 表示不限制次数。
- 监控变化：选择页面区域 XPath 后，刷新或点击后对比内容变化。
- 通知提醒：支持语音通知和浏览器系统弹窗通知。
- 自定义通知文案：弹窗通知内容支持按网址单独保存，未设置时使用默认文案。
- 系统通知标题：监控变化触发时，通知标题显示当前监测网页地址。
- 自动关闭通知：系统弹窗通知默认 3 秒后自动关闭。
- 悬浮窗口：可在网页右下角显示简洁控制窗，查看状态、倒计时、启动/停止刷新和重置间隔。
- 按网址缓存：监控 XPath、悬浮窗开关、弹窗通知文案均按当前网址关联保存。
- 活动标签页列表：可查看当前正在运行刷新任务的网页。
- 日志倒计时：插件日志页固定显示刷新倒计时，并记录启动、停止、刷新、异常和监控变化。

## 安装方式

1. 下载或克隆本仓库。
2. 打开 Chrome，访问 `chrome://extensions/`。
3. 开启右上角“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择本项目中的 `Auto Refresh/` 目录。
6. 在工具栏打开 Auto Refresh Pro 即可使用。

## 使用说明

### 1. 设置刷新间隔

在“间隔”页设置刷新间隔，支持手动输入秒数，也可以使用快捷预设，例如 `10s`、`30s`、`60s`、`2m`、`5m`。

如果需要限制刷新次数，可以设置“刷新上限”。设置为 `0` 时表示无限制。

### 2. 选择刷新模式

在“监控”页选择刷新模式：

- 全页刷新：到达倒计时后刷新整个页面。
- XPath 点击：到达倒计时后点击指定 XPath 元素。

使用 XPath 点击模式时，可以点击“选取”按钮，然后在网页中选择目标元素。插件会自动生成 XPath 并保存。

### 3. 开启监控变化

开启“监控变化”后，需要设置监控区域 XPath。

可以手动输入 XPath，也可以点击“选取”按钮在网页中选择需要监控的区域。插件会在刷新或 XPath 点击前后对比区域文本，如果内容发生变化，就触发通知。

### 4. 配置通知

监控变化支持两类通知：

- 语音通知：在目标网页环境中播放语音提醒。
- 弹窗通知：使用 Chrome 系统通知提醒，即使当前不在目标网页也能看到。

弹窗通知内容只有在开启“弹窗通知”开关后才可以编辑。自定义内容会按当前网址保存，不会影响其它网址。

默认通知文案：

```text
监控区域发生变化，请及时查看。
```

### 5. 使用悬浮窗口

在“间隔”页开启“悬浮窗口”后，网页右下角会出现简洁控制窗。

悬浮窗口提供：

- 运行状态
- 倒计时
- 启动/停止刷新
- 重置当前刷新间隔

悬浮窗口开关按网址缓存，不同网址之间互不影响。

### 6. 查看运行标签和日志

“标签”页会显示当前正在执行刷新任务的网页。

“日志”页会显示：

- 固定刷新倒计时
- 启动和停止记录
- 刷新完成记录
- 监控变化记录
- 异常信息

## 权限说明

插件使用以下 Chrome 权限：

- `storage`：保存刷新间隔、XPath、监控配置、通知文案和悬浮窗开关。
- `activeTab`：获取当前活动标签页并执行用户触发的操作。
- `scripting`：在目标页面中执行 XPath 文本读取、点击和 content script 注入。
- `tabs`：读取标签页状态、URL、标题，并维护多标签刷新任务。
- `alarms`：作为定时刷新任务的后台兜底调度。
- `notifications`：在监控变化时显示 Chrome 系统通知。

## 项目结构

```text
Auto Refresh/
|-- manifest.json      # Chrome 扩展配置
|-- background.js      # 后台 Service Worker，管理定时器、状态和通知
|-- content.js         # 网页脚本，负责 XPath、元素选取、页面提醒和悬浮窗
|-- popup.html         # 插件弹窗结构
|-- popup.css          # 插件弹窗样式
|-- popup.js           # 插件弹窗交互、配置缓存、日志和状态同步
|-- preview.html       # UI 预览页
|-- AGENTS.md          # 项目结构说明
`-- icons/             # 插件图标
```

## 开发说明

本项目不依赖前端框架和构建工具，直接以源码目录作为 Chrome 扩展加载目录。

常用检查命令：

```bash
node --check popup.js
node --check background.js
node --check content.js
```

## 隐私说明

Auto Refresh Pro 的配置数据存储在浏览器本地 `chrome.storage.local` 中。插件不会主动上传用户的网页内容、XPath、通知文案或刷新配置。

## 适用场景

- 实时行情页面自动刷新
- 新闻、公告、数据面板变化监控
- 需要定时点击页面按钮的内部系统
- 多标签页刷新任务管理
- 需要网页右下角快捷控制窗的刷新场景

## 注意事项

- 系统通知能否显示，取决于 Chrome 和操作系统通知权限。
- 部分网站可能限制脚本执行、跨域 iframe 或自动点击行为。
- 如果插件更新后网页中的悬浮窗或 content script 表现异常，建议重新加载扩展并刷新目标网页。
